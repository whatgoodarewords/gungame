# Live E2E findings post-4b deploy (Prime probe, 2026-07-19)

Evidence: headless Chromium vs live dev.sml.world/gg; bots clean 60s (0 err) — both bugs browser-only.

1. **P0 — pointer lock dead**: `WrongDocumentError: root document of this element not valid for pointer lock`. 4b's front-door rebuild replaced/reparented the canvas; RawInput holds the detached element. Fix: input binds to the LIVE canvas (acquire at lock time or rebind on mount), plus a regression assertion in the e2e probe (lock engages after join+click).
2. **P0 — browser WS drops ~8s in, after first style switch** (probe timeline: WS OPEN → in game → dev-grid alive → WS CLOSED → next style shows CONNECTION LOST). No console error, bots unaffected. Suspects: style-transaction rollback path; pump worker + backpressure hard-threshold interplay; bg-flag misfire. Needs instrumented repro (WS close code/reason surfaced to console + probe).

3. **P1 (owner-reported 2026-07-19) — z-fighting where ramps/objects meet the ground**: rapid flicker on coplanar faces. Root cause: map generators emit object base faces exactly at floor height (coplanar planes cannot be resolved by any depth precision). Fix in the generators, not the renderer: (a) sink every grounded object's base 0.02 m below the floor surface so side walls intersect cleanly and base faces are depth-occluded; (b) stop emitting never-visible bottom faces entirely (also saves triangles); (c) audit all four maps for any two coplanar faces from different primitives (validator rule: no two triangles within 1 mm parallel-coplanar overlap — make it a map-pipeline CI check so this class can never ship again); (d) belt-and-braces: enable logarithmic depth on the WebGL2 path if free, but geometry is the real fix.

4. **P0 (owner-reported, persists post-4c) — crouch-jump (+turning) still broken on live.** Likely compound: the confirmed pointer-lock P0 (turning dies when lock fails) × duck binding. MANDATORY diagNOSTIC with the fix: the dev panel gains an input inspector row (live button bits, lock state, last 5 key events) so any future input report is diagnosable from a screenshot. Test matrix for the fix: Shift+Space+A/D+mouse simultaneously on macOS Chrome/Safari — jump fires while ducked and turning, every combination.

5. **P0 CRITICAL (Prime log-mining 2026-07-19) — live server process crashed on uncaught FSM ProtocolError.** Evidence: docker logs show full process death + restart ("listening" twice); stack: ConnectionFsm.timedOut ← sweepTimeouts ← AuthoritativeLoop.step, "FSM time must be finite and monotonic" (packages/protocol/src/fsm.ts:44/59). Two fixes required: (a) root-cause the clock mismatch — the FSM's stored time vs the loop's sweep time come from different sources/moments (suspect: connection FSM stamped with a fresher now() than the loop's tick-start time; sweep then presents an older time → guard trips on innocent traffic like an uncleanly-closed tab); unify on ONE loop clock passed down, never re-sampled; (b) blast-radius boundary — per-connection errors inside step/sweep must close THAT connection (typed disconnect + log), never escape to kill the process; add a loop-level catch that quarantines the offending connection and continues the tick. Regression tests: uncleanly-dropped connection then sweep (no throw); an FSM forced into a non-monotonic call closes one connection while the room's other connections tick on. NOTE: this most likely explains the recurring mid-session WS drops (probe and owner reports) — server was dying and auto-restarting under us.
